# Data Engineering – SQL & PySpark Journey

## 🔹 Overview

This repository showcases my learning and hands-on implementation of data engineering concepts using SQL and PySpark. It covers data cleaning, transformation, analytical processing, and building end-to-end pipelines on real-world datasets.

---

## 🔹 Phases Covered

### 📌 SQL – Data Cleaning & Functions
- Handled NULL values and avoided calculation errors  
- Standardized text data using string functions  
- Worked with date functions for time-based analysis  
- Applied conditional logic for data categorization  

---

### 📌 SQL – Conditional Logic & Window Functions
- Used `CASE WHEN` for business rules  
- Implemented ranking using `ROW_NUMBER()`, `RANK()`, `DENSE_RANK()`  
- Applied `PARTITION BY` for grouped analysis  
- Performed advanced SQL-based analytics  

---

### 📌 PySpark – Data Cleaning & ETL
- Cleaned data using `dropna()`, `fillna()`, and filtering  
- Removed duplicates using `dropDuplicates()`  
- Built ETL pipeline (Extract → Transform → Load)  
- Validated data after transformations  

---

### 📌 PySpark – Analytical Transformations
- Joined multiple datasets  
- Created derived columns (e.g., revenue)  
- Applied aggregations and window functions  
- Generated insights like top customers and sales trends  
- Saved outputs in Parquet and CSV formats  

---

### 📌 PySpark – Regex & Data Transformation
- Extracted patterns using Regular Expressions  
- Cleaned and standardized text data  
- Handled NULL values effectively  
- Applied filtering, sorting, and deduplication  

---

### 📌 Databricks – End-to-End Pipeline (Olist Dataset)
- Worked with real-world multi-table dataset  
- Created fact table (`fact_orders`) using joins  
- Applied window functions for advanced analysis  
- Performed customer segmentation and CLV  
- Generated final reporting dataset
